# Lightdash MCP 用户使用说明（分析师版）

本文面向分析师、运营、业务同学。目标是：**少记工具名，直接拿结果**。

---

## 1. 你只需要记住这条原则

**先找现成图表；找不到再做自定义查询。**

推荐顺序：

1. 找项目：`lightdash_list_projects`
2. 找图表/看板：`lightdash_search_content`
3. 跑图表拿数：`lightdash_run_saved_chart`
4. 只有必要时才用高级查询：`lightdash_run_metric_query`

---

## 2. 三种常见场景

### 场景 A：我只想看某张图或某个看板的数据

你可以这样提问：

- “帮我找价格指数相关图表，并给我最近 12 个月数据”
- “查一下即饮茶相关的保存图表，先给结论”

系统会自动：

- 搜索内容 -> 读取图表参数 -> 运行图表 -> 输出结论

---

### 场景 B：没有现成图表，我要临时分析

你可以这样提问：

- “按三级类目看商品数 Top10”
- “按品牌看某类目销量，按降序”

系统会自动：

- 选择数据主题（explore）
- 识别维度/指标
- 尝试筛选和排序
- 返回可解释结果

---

### 场景 C：我想查明细/查表（高级）

你可以这样提问：

- “我要明细数据，不是图表汇总”
- “按日期和品牌导出明细结构”

说明：

- 如果当前能力不支持直接 SQL，会自动用可替代的 explore 查询路径完成
- 会明确告诉你“已替代查询”或“需要补充条件”

---

## 3. 高成功率提问模板（建议复制）

### 模板 1：查图表

“在项目 `<项目名或projectUuid>` 中，查 `<主题关键词>` 的已保存图表，返回 `<时间范围>` 的关键结论和3个核心数字。”

### 模板 2：临时分析

“在项目 `<projectUuid>` 中，按 `<维度>` 分析 `<指标>`，筛选 `<条件>`，返回 Top `<N>`，并解释口径。”

### 模板 3：逐步下钻

“先给总览，再给 Top10，再下钻到品牌/品类，不要一次返回太大结果。”

## 4. 常见问题（FAQ）

### Q1：为什么有时会提示字段错误？

A：通常是把维度当指标，或字段口径不匹配。系统会自动纠正一部分；仍失败时会自动回退到字段核对流程。

### Q2：为什么有时查询慢？

A：常见原因是范围太大。建议先查小范围（如 Top50、近12个月），确认后再扩大。

### Q3：为什么筛选会失败？

A：复杂筛选组合可能触发上游限制。建议先单条件（equals）成功后再叠加条件。

### Q4：我需要记住很多工具吗？

A：不需要。你只需按“先图表、后自定义”的顺序提需求即可。

---

## 5. 推荐工作流（团队统一）

1. 先给业务问题，不先给技术参数
2. 先要结论，再要明细
3. 先小范围验证，再扩大范围
4. 失败时让系统“换一种查法”，不要重复同一句

---

## 6. 可直接发给团队的使用口令

> 你是我的 Lightdash 数据助手。  
> 优先帮我找现成图表并直接出结论；找不到再做自定义查询。  
> 输出顺序固定为：结论 -> 关键数字 -> 口径说明。  
> 先小范围快速返回，不要一次给超大结果。
